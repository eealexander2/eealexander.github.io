Active Record's Duplicate Key Update

December 18, 2017

Recently, I learned about Active Record's on_duplicate_key_update. "on_duplicate_key_update" is part of Active Record's import method. The on_duplicate_key allows the user to specify which fields to update when a duplicate record is found during the import. If there is an existing record in Active Record's database, Active Record will only update the specified fieds in the existing record instead of creating an entirely new record.

How does ActiveRecord know how to identify duplicate records, you ask? Great question, I was thinking it too. You can let Active Record know how to identify duplicate records by passing in a field column to conflict_target. For example:

equities = Equity.all Equity.import {equities conflict_target: [:symbol], columns: [:description, :opol, :exchanges, :cfi]}

In this case, Active Record knows how to identify duplicate records by finding those that share the same "conflict target" attribute. In this case, if Active Record encounters a record that shares the same symbol as the imported record, it will update the existing record instead of creating a new one. How does Active Record know which fields to update? Another great question. To specify which fields to update upon finding a duplicate record, pass Active Record specific columns. In this example, the columns to update would be description, opol, exchanges, and cfi.

And there you have it! A quick overview of Active Record's "on_duplicate_key_update". Cheers!
